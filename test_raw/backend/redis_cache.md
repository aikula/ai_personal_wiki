# Redis Cache — Backend Infrastructure

Проект backend использует Redis 7.2 в качестве основного слоя кеширования.

## Установка и конфигурация

Redis разворачивается в Docker-контейнере. Базовая конфигурация:

```yaml
services:
  redis:
    image: redis:7.2-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    command: redis-server --save 60 1 --loglevel warning
```

Подключение из приложения через переменную окружения `REDIS_URL=redis://localhost:6379/0`.

## Использование в приложении

Redis используется для трёх задач:

1. **Кеширование сессий** — TTL 24 часа, ключ `session:{user_id}`
2. **Rate limiting** — скользящее окно 60 секунд, реализовано через Lua-скрипт
3. **Очередь фоновых задач** — через Celery с брокером Redis

Кеш инвалидируется явно при обновлении данных пользователя. Автоматическая инвалидация по TTL — запасной механизм.

## Мониторинг

Метрики снимаются через redis-exporter и отправляются в Prometheus.
Алерт настроен на `used_memory > 80%` от `maxmemory`.

## Ограничения

- `maxmemory` выставлен в 512MB
- Политика вытеснения: `allkeys-lru`
- Persistence отключён для кеш-инстанса (это нормально для кеша)

## Связанные компоненты

Для хранения основных данных используется PostgreSQL — он не конкурирует с Redis, они выполняют разные роли.
